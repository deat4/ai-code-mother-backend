## 1. LangChain4j TokenStream 错误

### 问题现象

```
IllegalConfigurationException: Illegal method return type: interface dev.langchain4j.service.TokenStream
    at dev.langchain4j.service.output.PojoOutputParser.validateJsonStructure
```

### 根因分析

1. 项目自定义了 `dev/langchain4j/service/TokenStream.java` 接口，试图扩展官方功能
2. LangChain4j 在 `AiServices` 构建代理时，使用 `==`（身份比较）验证返回类型
3. 自定义接口与官方 JAR 中的 `TokenStream.class` 不是同一个类对象
4. 验证失败，抛出异常

### 解决方案

**删除自定义接口，使用官方 API**

```
删除文件:
- src/main/java/dev/langchain4j/service/TokenStream.java
- src/main/java/dev/langchain4j/service/TokenStreamExample.java
```

**修改 AiCodeGeneratorService.java**：
```java
// 使用官方 @SystemMessage 注解加载系统提示词
@SystemMessage(fromResource = "prompt/codegen-vue-project-system-prompt.txt")
TokenStream generateVueProjectCodeStream(@MemoryId long appId, @UserMessage String userMessage);
```

**修改 AiCodeGeneratorFacade.java**：
- 移除 `.systemMessage()` 手动设置
- 使用官方 `TokenStream` API 处理流式响应

### 经验教训

| 教训 | 说明 |
|------|------|
| **不要覆盖框架核心类** | LangChain4j 使用身份比较 (`==`) 而非 `equals()`，自定义类无法通过验证 |
| **优先使用官方扩展点** | 框架提供的注解、接口是首选，避免魔改源码 |
| **理解框架设计意图** | `TokenStream` 是流式输出专用接口，框架内部有特殊处理逻辑 |

## 2. SQL 语法错误 - Unknown column

### 问题现象

```
Unknown column 'versionNumber' in 'order clause'
```

### 根因分析

1. MyBatis-Flex 的 `orderBy()` 方法使用字符串参数
2. 开发者传入 Java 字段名 `versionNumber`
3. 数据库实际列名是 `version_number`（下划线命名）
4. SQL 执行时找不到 `versionNumber` 列

### 解决方案

**AppVersionServiceImpl.java**：
```java
// 错误写法
orderBy("versionNumber", desc);

// 正确写法
orderBy("version_number", desc);
```

### 经验教训

| 教训 | 说明 |
|------|------|
| **区分 Java 字段名和数据库列名** | MyBatis-Flex 的原生 SQL 方法需要使用数据库列名 |
| **使用驼峰命名自动映射** | 配置 `map-underscore-to-camel-case: true` 让 MyBatis 自动转换 |
| **优先使用类型安全的 API** | 如 `QueryWrapper.orderBy(AppVersion::getVersionNumber, false)` |

## 3. 数据库字段溢出 - Data too long

### 问题现象

```
Data too long for column 'message' at row 1
```

### 根因分析

1. `chat_history.message` 字段类型为 `VARCHAR`
2. AI 生成的代码内容可能很长（数万字符）
3. VARCHAR 有长度限制，无法存储大文本

### 解决方案

**数据库迁移 SQL**：
```sql
ALTER TABLE chat_history MODIFY COLUMN message LONGTEXT;
```

**实体类调整**：
```java
// ChatHistory.java - message 字段类型保持 String
// MyBatis-Flex 会自动映射 LONGTEXT 到 String
private String message;
```

### 经验教训

| 教训 | 说明 |
|------|------|
| **评估字段最大长度** | AI 生成内容、用户输入等动态内容应使用 LONGTEXT |
| **数据库设计要考虑极端情况** | 测试时使用简单数据，生产环境可能遇到大文本 |
| **添加字段长度日志** | 存储失败时记录内容长度，便于排查 |

## 4. SSE 流式响应错误处理

### 问题现象

```
No converter for [com.zkf.aicodemother.common.BaseResponse] with Content-Type 'text/event-stream'
```

### 根因分析

1. SSE 流式接口 `Content-Type: text/event-stream`
2. 发生错误时调用 `sink.error(exception)` 试图返回 JSON
3. Spring MVC 无法将 JSON 对象序列化为 SSE 格式
4. 导致 Content-Type 冲突

### 解决方案

**AiCodeGeneratorFacade.java**：
```java
// 错误做法
sink.error(new BusinessException(ErrorCode.INSUFFICIENT_BALANCE));

// 正确做法 - 将错误作为 SSE 事件发送
private void sendErrorAsEvent(FluxSink<String> sink, Throwable error) {
    String errorMessage = error.getMessage();
    
    // 特殊处理余额不足错误
    if (errorMessage.contains("Insufficient Balance")) {
        String errorJson = String.format(
            "{\"type\":\"error\",\"code\":\"INSUFFICIENT_BALANCE\",\"message\":\"%s\"}",
            "余额不足，请充值后重试"
        );
        sink.next(errorJson);
        sink.complete();
        return;
    }
    
    // 其他错误
    String errorJson = String.format(
        "{\"type\":\"error\",\"message\":\"%s\"}",
        errorMessage.replace("\"", "\\\"")
    );
    sink.next(errorJson);
    sink.complete();
}
```

### 经验教训

| 教训 | 说明 |
|------|------|
| **SSE 流不能中断** | 一旦开始流式输出，必须以 `data:` 格式发送所有内容 |
| **错误也是事件** | 将错误包装为 SSE 事件发送，前端统一处理 |
| **保持响应格式一致** | 无论成功还是失败，都使用相同的 JSON 结构 |

## 5. Vue 项目预览 404

### 问题现象

```
GET http://localhost:8123/src/main.js 404
```

### 根因分析

1. Vue 项目执行 `npm run build` 后，输出在 `dist/` 目录
2. `StaticResourceController.previewGeneratedCode()` 直接读取项目根目录
3. 没有考虑 Vue 项目的 `dist/` 前缀
4. 请求 `/src/main.js` 实际应读取 `dist/src/main.js`

### 解决方案

**StaticResourceController.java**：
```java
@GetMapping("/preview/{codeGenTypeAppId}/**")
public void previewGeneratedCode(
        @PathVariable String codeGenTypeAppId,
        HttpServletRequest request,
        HttpServletResponse response) {
    
    String requestPath = request.getRequestURI();
    String resourcePath = requestPath.substring(previewPrefix.length());
    
    // Vue 项目特殊处理：优先查找 dist 目录
    if (codeGenTypeAppId.startsWith("VUE_PROJECT_")) {
        Path distPath = Paths.get(projectDir, "dist", resourcePath);
        if (Files.exists(distPath)) {
            writeResource(response, distPath);
            return;
        }
    }
    
    // 默认路径
    Path filePath = Paths.get(projectDir, resourcePath);
    writeResource(response, filePath);
}
```

### 经验教训

| 教训 | 说明 |
|------|------|
| **理解构建工具输出结构** | Vue/React 项目构建后输出在 `dist/` 或 `build/` 目录 |
| **区分开发环境和生产环境路径** | 开发时 `src/main.js`，构建后 `dist/assets/index-xxx.js` |
| **添加路径日志** | 记录请求路径和实际读取路径，便于排查 404 |

## 6. 枚举值大小写不一致

### 问题现象

前端发送 `"vue_project"`，后端枚举定义 `"VUE_PROJECT"`，导致匹配失败。

### 根因分析

1. 后端 `CodeGenTypeEnum` 定义为 `VUE_PROJECT`（大写）
2. 前端传递 `vue_project`（小写下划线）
3. 字符串比较失败，功能异常

### 解决方案

**统一使用大写**：

**CodeGenTypeEnum.java**：
```java
public enum CodeGenTypeEnum {
    HTML("HTML"),
    MULTI_FILE("MULTI_FILE"),
    VUE_PROJECT("VUE_PROJECT");
    
    // 序列化时输出大写
    @JsonValue
    public String toValue() {
        return this.name();
    }
}
```

**前端对应修改**：
```typescript
// 统一使用大写
const CODE_GEN_TYPES = {
  HTML: 'HTML',
  MULTI_FILE: 'MULTI_FILE',
  VUE_PROJECT: 'VUE_PROJECT'
} as const;
```

**StaticResourceController.java** - 路径前缀统一：
```java
// 修改前
if (codeGenTypeAppId.startsWith("vue_project_"))

// 修改后
if (codeGenTypeAppId.startsWith("VUE_PROJECT_"))
```

### 经验教训

| 教训 | 说明 |
|------|------|
| **前后端枚举格式统一** | 约定一种格式（大写/小写），全局遵守 |
| **使用 @JsonValue 控制序列化** | 确保输出格式一致，避免隐式转换 |
| **接口文档明确格式要求** | Swagger/OpenAPI 注释说明枚举值格式 |

## 7. JSON 格式配置冲突

### 问题现象

DeepSeek 模型返回格式异常，工具调用参数 JSON 解析失败。

### 根因分析

1. `application-location.yml` 配置了 `response-format: json_object`
2. DeepSeek 要求：使用 `json_object` 格式时，提示词必须包含 `json` 关键词
3. 代码生成提示词不满足此要求
4. 模型输出格式不一致，导致解析失败

### 解决方案

**application-location.yml**：
```yaml
# 移除 JSON 格式强制配置
# response-format: json_object  # 删除
# strict-json-schema: true      # 删除

# 保留模型参数
max-tokens: 4096
temperature: 0.3
```

### 经验教训

| 教训 | 说明 |
|------|------|
| **阅读模型 API 文档** | 不同模型对配置参数有不同的限制和要求 |
| **JSON 格式配置需谨慎** | 强制 JSON 输出可能干扰工具调用和流式响应 |
| **测试各种场景** | 简单问答、代码生成、工具调用都要验证 |

## 8. Multi-File 代码生成缺少文件

### 问题现象

- `styles.css` 为空
- `script.js` 返回 404
- 页面样式简陋

### 根因分析

1. **AI 只输出 HTML 代码块**，没有输出 CSS 和 JS
2. DeepSeek 模型在简单需求时倾向于输出单个 HTML 文件
3. 原解析器只检测代码块，未提取 HTML 内联样式和脚本
4. 文件保存器跳过空内容文件

### 解决方案

#### 8.1 系统提示词强化

**codegen-multi-file-system-prompt.txt**：
```
⚠️ 绝对规则：必须输出三个代码块
即使最简单的页面，也必须分别输出 HTML、CSS、JS！

响应格式：
​```html
<!-- HTML 内容 -->
​```

​```css
/* CSS 样式 */
​```

​```javascript
// JS 交互
​```

输出检查清单：
- [ ] 是否有 HTML 代码块？
- [ ] 是否有 CSS 代码块？
- [ ] 是否有 JS 代码块？
- [ ] HTML 是否正确引用了 styles.css 和 script.js？
```

#### 8.2 解析器兜底逻辑

**MultiFileCodeParser.java**：
```java
/**
 * 从 HTML 中提取内联样式和脚本
 */
private void extractInlineStylesAndScripts(String html, MultiFileCodeResult result) {
    // 提取 <style> 标签内容
    Pattern stylePattern = Pattern.compile("<style[^>]*>([\\s\\S]*?)</style>");
    Matcher styleMatcher = stylePattern.matcher(html);
    StringBuilder cssBuilder = new StringBuilder();
    while (styleMatcher.find()) {
        cssBuilder.append(styleMatcher.group(1)).append("\n");
    }
    if (cssBuilder.length() > 0) {
        result.setCss(cssBuilder.toString());
    }
    
    // 提取 <script> 标签内容（排除外部脚本）
    Pattern scriptPattern = Pattern.compile(
        "<script(?![^>]*src=)[^>]*>([\\s\\S]*?)</script>"
    );
    Matcher scriptMatcher = scriptPattern.matcher(html);
    StringBuilder jsBuilder = new StringBuilder();
    while (scriptMatcher.find()) {
        jsBuilder.append(scriptMatcher.group(1)).append("\n");
    }
    if (jsBuilder.length() > 0) {
        result.setJs(jsBuilder.toString());
    }
}
```

#### 8.3 智能生成默认 CSS/JS

**MultiFileCodeFileSaverTemplate.java**：
```java
@Override
protected void saveFiles(MultiFileCodeResult result, String outputDir) {
    String html = result.getHtml();
    String css = StrUtil.blankToDefault(result.getCss(), () -> generateDefaultCss(html));
    String js = StrUtil.blankToDefault(result.getJs(), () -> generateDefaultJs(html));
    
    writeToFile(outputDir + "/index.html", html);
    writeToFile(outputDir + "/styles.css", css);
    writeToFile(outputDir + "/script.js", js);
}

/**
 * 根据 HTML 内容智能生成 CSS
 */
private String generateDefaultCss(String html) {
    StringBuilder css = new StringBuilder();
    
    // 基础重置
    css.append("*, *::before, *::after { box-sizing: border-box; }\n");
    css.append("body { font-family: system-ui, sans-serif; line-height: 1.6; }\n\n");
    
    // 检测 HTML 元素并添加样式
    if (html.contains("<h1") || html.contains("<h2")) {
        css.append("h1, h2, h3 { margin: 1rem 0; }\n");
    }
    if (html.contains("<button")) {
        css.append("button { padding: 0.5rem 1rem; border-radius: 4px; cursor: pointer; }\n");
        css.append("button:hover { opacity: 0.9; }\n");
    }
    if (html.contains("<input") || html.contains("<form")) {
        css.append("input, textarea { width: 100%; padding: 0.5rem; border: 1px solid #ccc; border-radius: 4px; }\n");
        css.append("input:focus { outline: none; border-color: #007bff; }\n");
    }
    
    // 工具类
    css.append("\n/* Utilities */\n");
    css.append(".text-center { text-align: center; }\n");
    css.append(".mt-1 { margin-top: 0.5rem; }\n");
    css.append(".mt-2 { margin-top: 1rem; }\n");
    
    return css.toString();
}
```

### 经验教训

| 教训 | 说明 |
|------|------|
| **AI 输出不可控** | 即使提示词明确要求，模型仍可能省略部分内容 |
| **多层防护** | 系统提示词 → 解析器兜底 → 默认生成，确保最终结果可用 |
| **智能默认值** | 根据上下文（HTML 内容）生成相关默认值，而非固定模板 |
| **日志调试** | 添加 DEBUG 日志记录 AI 原始输出和解析结果 |

## 9. 版本号并发冲突

### 问题现象

```
Duplicate entry '391477244803891200-1' for key 'uk_app_version'
```

### 根因分析

```
时间线：
19:31:35.475 - AI 自动创建版本成功 (versionNumber=1)
19:31:35.523 - 前端手动请求创建版本失败 (Duplicate)
```

1. **并发请求**：AI 代码生成完成后自动创建版本，前端同时手动保存版本
2. **乐观锁失效**：两个请求都读取 `app.currentVersion=0`，都尝试创建版本 1
3. **唯一键冲突**：`uk_app_version(app_id, version_number)` 约束导致插入失败

### 解决方案

**AppVersionServiceImpl.java**：
```java
@Override
public Long createVersion(AppVersionCreateRequest request) {
    // 原代码：从 app 实体读取 currentVersion（并发不安全）
    // App app = appMapper.selectOneById(request.getAppId());
    // Integer currentVersion = app.getCurrentVersion();
    
    // 修复：从数据库查询最大版本号
    QueryWrapper maxVersionWrapper = QueryWrapper.create()
        .eq("app_id", request.getAppId())
        .select("MAX(version_number) as version_number");
    AppVersion maxVersion = appVersionMapper.selectOneByQuery(maxVersionWrapper);
    
    Integer currentVersion = (maxVersion != null && maxVersion.getVersionNumber() != null)
        ? maxVersion.getVersionNumber()
        : 0;
    
    Integer newVersionNumber = currentVersion + 1;
    
    // 创建版本记录...
}
```

### 经验教训

| 教训 | 说明 |
|------|------|
| **并发场景要考虑** | AI 自动操作和用户手动操作可能同时发生 |
| **数据库是唯一真相** | 从数据库查询最新状态，而非依赖内存中的实体 |
| **使用数据库约束** | 唯一键约束是最后的防护，避免脏数据 |
| **幂等设计** | 重复请求应返回相同结果，而非报错 |

## 10. AppVO 字段增强

### 问题现象

前端需要展示应用描述、当前版本号、总版本数、预览链接，但后端 VO 缺少这些字段。

### 解决方案

#### 10.1 实体类修改

**App.java**：
```java
/**
 * 应用描述
 */
private String appDesc;
```

**AppVO.java**：
```java
/**
 * 应用描述
 */
private String appDesc;

/**
 * 当前版本号
 */
private Integer currentVersion;

/**
 * 总版本数
 */
private Integer totalVersions;

/**
 * 预览链接
 */
private String previewUrl;
```

#### 10.2 数据库迁移

```sql
ALTER TABLE app ADD COLUMN app_desc VARCHAR(500) COMMENT '应用描述' AFTER app_name;
```

#### 10.3 服务层填充

**AppServiceImpl.java**：
```java
@Override
public AppVO getAppVO(App app) {
    AppVO vo = new AppVO();
    BeanUtils.copyProperties(app, vo);
    
    // 填充版本信息
    Long totalVersions = appVersionMapper.selectCountByQuery(
        QueryWrapper.create().eq("app_id", app.getId())
    );
    vo.setTotalVersions(totalVersions.intValue());
    
    // 填充预览链接
    if (app.getCurrentVersion() > 0) {
        String codeGenType = app.getCodeGenType();
        String prefix = codeGenType.equals(CodeGenTypeEnum.VUE_PROJECT.getValue()) 
            ? "VUE_PROJECT_" 
            : "MULTI_FILE_";
        vo.setPreviewUrl("/preview/" + prefix + app.getId() + "/index.html");
    }
    
    return vo;
}
```

### 经验教训

| 教训 | 说明 |
|------|------|
| **VO 设计要考虑前端需求** | 不要简单复制实体字段，考虑展示所需的数据 |
| **计算字段放在 VO** | 版本数、预览链接等计算结果属于 VO，不是实体 |
| **提供完整迁移脚本** | 数据库变更要提供 ALTER 语句，确保环境一致 |
