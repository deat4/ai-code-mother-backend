## 1. 智能路由功能

### 功能说明

用户创建应用时，**无需手动选择代码生成类型**。后端会根据用户输入的需求描述，自动选择最合适的代码生成方案：

| 类型 | 适用场景 | 成本 |
|------|----------|------|
| HTML | 简单静态页面、落地页、介绍页 | 低 |
| MULTI_FILE | 多页面、有交互效果的静态站点 | 中 |
| VUE_PROJECT | 复杂应用、管理系统、CRUD | 高 |

### API 变更

**创建应用接口**：`POST /api/app/add`

**请求参数变更**：

```typescript
// 之前：codeGenType 必填
{
  "appName": "我的应用",
  "initPrompt": "做一个个人介绍页面",
  "codeGenType": "HTML"  // 必填
}

// 现在：codeGenType 可选，不传则自动路由
{
  "appName": "我的应用",
  "initPrompt": "做一个个人介绍页面"
  // codeGenType 可选，不传则 AI 自动选择
}
```

**响应不变**：
```json
{
  "code": 0,
  "data": 123456789,  // 应用 ID
  "message": "ok"
}
```

### 前端更新建议

1. **移除代码类型选择器**（推荐）
   - 简化用户体验，让 AI 自动判断
   - 只在高级选项中保留手动选择功能

2. **保留手动选择**（可选）
   - 如果保留选择器，需说明"不选择则自动判断"
   - 枚举值统一使用**大写格式**：`HTML`、`MULTI_FILE`、`VUE_PROJECT`

3. **枚举值格式要求**
   ```typescript
   // 正确 ✅
   const CODE_GEN_TYPES = {
     HTML: 'HTML',
     MULTI_FILE: 'MULTI_FILE',
     VUE_PROJECT: 'VUE_PROJECT'
   } as const;

   // 错误 ❌（旧格式）
   const CODE_GEN_TYPES = {
     HTML: 'html',
     MULTI_FILE: 'multi_file',
     VUE_PROJECT: 'vue_project'
   };
   ```

## 2. 代码下载功能

### 功能说明

用户可以下载自己创建的应用源代码，打包为 ZIP 文件。

### 新增接口

**下载应用代码**：`GET /api/app/download/{appId}`

| 项目 | 说明 |
|------|------|
| **方法** | GET |
| **路径** | `/api/app/download/{appId}` |
| **权限** | 仅应用创建者可下载 |
| **响应** | ZIP 文件流 |

**请求示例**：
```typescript
// 前端调用
const downloadCode = async (appId: number) => {
  window.open(`/api/app/download/${appId}`, '_blank');
};

// 或使用 axios 下载
const downloadCode = async (appId: number) => {
  const response = await axios.get(`/api/app/download/${appId}`, {
    responseType: 'blob'
  });
  
  // 创建下载链接
  const url = window.URL.createObjectURL(new Blob([response.data]));
  const link = document.createElement('a');
  link.href = url;
  link.setAttribute('download', `${appId}.zip`);
  document.body.appendChild(link);
  link.click();
  link.remove();
};
```

**错误响应**：
```json
{
  "code": 40100,
  "message": "无权限下载该应用代码",
  "data": null
}
```

**注意**：
- 只有应用创建者可以下载代码
- 代码目录不存在时返回 404 错误
- 下载的 ZIP 已过滤 `node_modules`、`dist`、`.git` 等目录

### 前端更新建议

1. **应用列表/详情页添加下载按钮**
   ```tsx
   <Button onClick={() => downloadCode(app.id)}>
     下载代码
   </Button>
   ```

2. **下载前检查应用状态**
   - 确保应用已生成代码
   - 提示用户仅可下载自己创建的应用

## 3. 封面图自动生成

### 功能说明

应用部署后，后端会**自动生成网页截图作为封面图**。

### 流程说明

```
用户点击部署 → 部署成功 → 异步生成截图 → 上传到 COS → 更新应用封面
```

### API 变更

**部署应用接口**：`POST /api/app/deploy`

响应不变，但部署成功后会自动生成封面图。

**获取应用详情**：`GET /api/app/{id}`

**响应新增字段**：
```json
{
  "code": 0,
  "data": {
    "id": 123456789,
    "appName": "我的应用",
    "cover": "https://your-cos-host.com/screenshots/2026/03/20/abc123_compressed.jpg",
    // ... 其他字段
  }
}
```

### 前端更新建议

1. **展示封面图**
   ```tsx
   <img 
     src={app.cover || DEFAULT_COVER} 
     alt={app.appName}
     onError={(e) => {
       e.target.src = DEFAULT_COVER;  // 加载失败时显示默认图
     }}
   />
   ```

2. **默认封面降级**
   - 封面生成可能失败（AI 服务异常、网络问题等）
   - 前端应设置默认封面图
   - `onError` 事件处理加载失败

3. **封面图更新时机**
   - 部署成功后，封面图会在几秒内生成
   - 如需立即显示，可在部署成功后轮询获取最新封面

## 4. 前端更新要点

### 4.1 枚举值统一大写

**影响的枚举**：
- `codeGenType`: `HTML`、`MULTI_FILE`、`VUE_PROJECT`

**修改示例**：
```typescript
// types/app.ts
export enum CodeGenType {
  HTML = 'HTML',
  MULTI_FILE = 'MULTI_FILE',
  VUE_PROJECT = 'VUE_PROJECT'
}

// 请求时
const createApp = async (data) => {
  // 不传 codeGenType，让后端自动路由
  return axios.post('/api/app/add', {
    appName: data.appName,
    initPrompt: data.initPrompt
    // codeGenType 可选
  });
};
```

### 4.2 新增下载功能

**API 调用**：
```typescript
// services/app.ts
export const downloadAppCode = (appId: number) => {
  window.open(`/api/app/download/${appId}`, '_blank');
};
```

**UI 组件**：
```tsx
// components/AppCard.tsx
<Button 
  icon={<DownloadOutlined />}
  onClick={() => downloadAppCode(app.id)}
>
  下载代码
</Button>
```

### 4.3 封面图展示

**默认封面**：
```typescript
const DEFAULT_COVER = 'https://via.placeholder.com/800x600?text=No+Preview';
```

**封面组件**：
```tsx
<img 
  src={app.cover || DEFAULT_COVER}
  alt={app.appName}
  className="app-cover"
  onError={(e) => {
    e.target.src = DEFAULT_COVER;
  }}
/>
```

### 4.4 AppVO 字段变更

| 字段 | 类型 | 说明 |
|------|------|------|
| `cover` | string | 封面图 URL（可能为空，需默认图） |
| `appDesc` | string | 应用描述（新增） |
| `currentVersion` | number | 当前版本号（新增） |
| `totalVersions` | number | 总版本数（新增） |
| `previewUrl` | string | 预览链接（新增） |

## 6. 常见问题

### Q1: 智能路由会不会选错类型？

A: AI 会根据用户需求判断，但可能存在偏差。如果用户对类型有明确要求，仍可在请求中手动指定 `codeGenType`。

### Q2: 封面图生成失败怎么办？

A: 后端会捕获异常并记录日志，不影响部署流程。前端应设置默认封面图作为降级方案。

### Q3: 下载代码包含哪些文件？

A: 下载的是源代码目录，已自动过滤：
- `node_modules`、`dist`、`build`
- `.git`、`.idea`、`.vscode`
- `.env`、`.DS_Store`
- `.log`、`.tmp`、`.cache`

### Q4: 非创建者可以下载代码吗？

A: 不可以，返回权限错误。前端应仅对创建者显示下载按钮。
