## 一、项目概述

### 1.1 项目定位

**AI Code Mother** 是一个智能代码生成平台后端服务，通过 AI 能力实现前端代码的自动化生成、版本管理和部署。用户只需描述需求，AI 即可生成可运行的 Web 应用代码。

### 1.2 核心能力

| 能力 | 描述 |
|------|------|
| **智能代码生成** | 根据用户自然语言描述，生成 HTML/CSS/JS 或完整 Vue 项目 |
| **流式实时输出** | SSE 流式传输，实时展示 AI 生成过程 |
| **对话记忆** | 支持多轮对话，AI 能记住上下文持续优化代码 |
| **版本管理** | 自动记录每次生成的版本，支持对比和回退 |
| **一键部署** | 生成的代码可直接部署并提供访问 URL |
| **智能路由** | AI 自动判断用户需求，选择最优生成模式 |
| **工具调用** | AI 可直接操作文件系统，实现复杂项目生成 |
| **自动验收** | 生成完成后自动校验代码质量，结构化反馈问题 |
| **任务追踪** | 完整的生成任务状态流转、日志记录、进度查询 |
| **封面自动生成** | 部署后自动截图生成应用封面 |

### 1.3 三种代码生成模式

| 模式 | 适用场景 | AI 工具 | 成本 |
|------|----------|---------|------|
| **HTML** | 简单静态页面、落地页、介绍页 | 无 | 低 |
| **MULTI_FILE** | 多页面、有交互效果的静态站点 | 无 | 中 |
| **VUE_PROJECT** | 复杂应用、管理系统、多页面应用 | 文件读写工具集 | 高 |

## 三、核心功能详解

### 3.1 AI 代码生成流程

#### 3.1.1 HTML/MULTI_FILE 模式流程

```
用户消息 → AppController.chatToGenCode()
         → AppService.chatToGenCode()
         → AiCodeGeneratorFacade.generateAndSaveCodeStream()
         → AiCodeGeneratorServiceFactory.getAiCodeGeneratorService()
         → AiCodeGeneratorService.generateHtmlCodeStream/generateMultiFileCodeStream()
         → Flux<String> 流式返回
         → processCodeStream() 收集完整内容
         → CodeParserExecutor.executeParser() 解析代码
         → CodeFileSaverExecutor.executeSaver() 保存文件
         → AppVersionService.createVersion() 创建版本记录
         → ScreenshotService 异步生成封面
```

#### 3.1.2 VUE_PROJECT 模式流程

```
用户消息 → AppController.chatToGenCode()
         → AppService.chatToGenCode()
         → 判断场景: CREATION / MODIFICATION
         → AiCodeGeneratorServiceFactory.getVueProjectCreatorService() 或 getVueProjectModifierService()
         → AiCodeCreator.generateCodeStream() 或 AiCodeModifier.updateCodeStream()
         → TokenStream → JsonMessageStreamHandler.handle()
         → 解析 JSON 消息:
            ├─ AI_RESPONSE: 直接输出文本
            ├─ TOOL_REQUEST: 显示工具调用信息
            └─ TOOL_EXECUTED: 执行工具并返回结果
         → 工具执行（FileWriteTool 等）
         → VueProjectBuilder.buildProjectAsyncByAppId() npm 构建
         → ChatHistoryOriginalService.batchAddMessages() 保存原始对话
         → ScreenshotService 异步生成封面
```

### 3.2 自动验收流程

#### 3.2.1 任务状态流转

**HTML/MULTI_FILE 流程**：
```
PENDING → RUNNING/INIT → GENERATING → VALIDATING → DONE + SUCCESS
失败路径: GENERATING → FAILED | VALIDATING → FAILED
```

**VUE_PROJECT 流程**：
```
PENDING → RUNNING/INIT → GENERATING → VALIDATING → BUILDING → DONE + SUCCESS
失败路径: GENERATING → FAILED | VALIDATING → FAILED | BUILDING → FAILED
```

#### 3.2.2 校验结果语义规则（重要）

| 字段 | 定义 | 说明 |
|------|------|------|
| `passed` | 只看是否有 ERROR | 有任何 ERROR 则 passed=false |
| `issueCount` | 只统计 ERROR 数量 | 用于失败统计和自动修复优先级 |
| `warningCount` | 只统计 WARN 数量 | 前端可展示"校验通过但有警告" |
| `summary` | 包含警告信息 | 例："HTML 校验通过，存在 2 个警告" |

**前端展示建议**：
- `passed=true, issueCount=0, warningCount=0` → "校验通过"
- `passed=true, issueCount=0, warningCount>0` → "校验通过，有 X 个警告"
- `passed=false` → "校验失败：{summary}"

#### 3.2.3 校验规则清单

| 类型 | 规则 | 失败摘要示例 |
|------|------|-------------|
| **HTML** | index.html 存在 | "入口文件 index.html 不存在" |
| | 文件非空 (>100字节) | "index.html 文件内容为空或过小" |
| | 无 markdown 包裹 | "文件中残留 markdown 代码块标记" |
| | 无占位符 (TODO/placeholder) | "文件中存在占位符" |
| | 必要标签存在 (<html>, <body>) | "HTML 结构不完整，缺少 <html> 标签" |
| **MULTI_FILE** | index.html 存在 | "入口文件 index.html 不存在" |
| | 至少 2 个有效文件 | "有效文件数量不足，至少需要 2 个文件" |
| | 无空文件 | "文件 xxx 内容过小" |
| | 无 markdown 残留 | "文件 xxx 中残留 markdown 代码块标记" |
| | 资源引用路径正常 | "HTML 中存在异常的资源引用路径" |
| **VUE_PROJECT** | package.json 存在 | "package.json 不存在" |
| | build 脚本配置 | "package.json 缺少 build 脚本" |
| | src/main.js/ts 存在 | "入口文件 main.js/main.ts 不存在" |
| | src/App.vue 存在 | "App.vue 不存在" |
| | npm install 成功 | "构建失败: npm install 失败" |
| | npm run build 成功 | "构建失败: npm run build 失败" |

#### 3.2.3 校验结果结构

```java
public class ValidationResult {
    private boolean passed;          // 是否通过
    private String summary;          // 摘要（可读描述）
    private String codeGenType;      // 代码类型
    private String stage;            // 阶段 (VALIDATING/BUILDING)
    private List<ValidationIssue> issues;  // 问题列表
    private BuildValidationResult buildResult; // 构建结果（Vue）
    private Map<String, Object> extra;      // 扩展信息

public class ValidationIssue {
    private String type;      // STRUCTURE / CONTENT / BUILD / CONFIG
    private String severity;  // INFO / WARN / ERROR
    private String filePath;  // 相关文件路径
    private String ruleCode;  // 规则编码 (INDEX_MISSING/NPM_BUILD_FAILED等)
    private String message;   // 错误描述
    private String suggestion;// 修复建议
}

public class BuildValidationResult {
    private boolean installSuccess;  // npm install 成功
    private boolean buildSuccess;    // npm run build 成功
    private Integer installExitCode; // install 退出码
    private Integer buildExitCode;   // build 退出码
    private String installLog;       // install 日志
    private String buildLog;         // build 日志
    private List<String> keyErrors;  // 关键错误提取
    private Long installDurationMs;  // install 耗时
    private Long buildDurationMs;    // build 耗时
}
```

`AiCodeGeneratorServiceFactory` 是核心组件，管理 AI 服务实例的创建和缓存：

```java
// 三层缓存架构
public class AiCodeGeneratorServiceFactory {
    // HTML/MULTI_FILE 服务缓存
    private final Cache<String, AiCodeGeneratorService> serviceCache =
        Caffeine.newBuilder()
            .maximumSize(1000)
            .expireAfterWrite(Duration.ofMinutes(30))
            .expireAfterAccess(Duration.ofMinutes(10))
            .build();

    // Vue 创建服务缓存
    private final Cache<String, AiCodeCreator> creatorServiceCache;

    // Vue 修改服务缓存
    private final Cache<String, AiCodeModifier> modifierServiceCache;
}
```

**缓存策略**：
- 最大缓存 1000 个实例
- 写入后 30 分钟过期
- 访问后 10 分钟过期
- 每个 appId 隔离对话记忆

**多例模式解决并发阻塞**：
```java
// 每次创建服务时获取新的 StreamingChatModel 实例
StreamingChatModel streamingChatModel = SpringContextUtil.getBean(
    "streamingChatModelPrototype", StreamingChatModel.class);
```

### 3.3 对话记忆双轨制

为解决 AI 工具调用信息丢失问题，采用双轨制存储：

| 表 | 存储内容 | 用途 |
|----|----------|------|
| `chat_history` | 格式化文本 | 前端展示、用户查看 |
| `chat_history_original` | 原始 JSON | AI 记忆恢复、工具调用信息 |

**工作原理**：

```
VUE_PROJECT 生成过程中:
├─ 收集格式化文本 → chat_historyStringBuilder
├─ 收集原始 JSON → originalMessages 列表
└─ 完成后批量保存:
    ├─ chat_history: 前端展示用
    └─ chat_history_original: AI 记忆恢复用
```

**记忆恢复流程**：
```java
// Vue 项目从原始历史表加载完整工具调用记忆
MessageWindowChatMemory chatMemory = MessageWindowChatMemory.builder()
    .id(appId)
    .chatMemoryStore(chatMemoryStore)
    .maxMessages(60)  // Vue 项目需要更多消息窗口
    .build();

chatHistoryOriginalService.loadOriginalHistoryToMemory(appId, chatMemory, 60);
```

### 6.1 Spring 多例模式解决并发阻塞

```java
@Configuration
public class StreamingChatModelConfig {

    @Bean
    @Scope("prototype")  // 多例模式
    public StreamingChatModel streamingChatModelPrototype() {
        return OpenAiStreamingChatModel.builder()
                .baseUrl(baseUrl)
                .modelName(modelName)
                .build();
    }
}

// 工厂类每次获取新实例
StreamingChatModel model = SpringContextUtil.getBean(
    "streamingChatModelPrototype", StreamingChatModel.class);
```
