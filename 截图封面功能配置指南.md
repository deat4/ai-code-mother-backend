# 截图封面功能配置指南

## 功能概述

截图封面功能用于在代码生成完成或应用部署后，自动截图并上传到对象存储，作为应用的封面图片。

## 触发时机

1. **代码生成完成后** - 异步截图预览页面
2. **应用部署后** - 异步截图已部署的应用页面

## 必需配置

### 1. 腾讯云 COS 配置

在 `application.yml` 或外部配置文件中添加：

```yaml
cos:
  client:
    secret-id: your-secret-id
    secret-key: your-secret-key
    region: ap-shanghai  # 根据实际情况填写
    bucket: your-bucket-name-12345678  # 格式: bucketname-appid
    host: https://your-bucket-name-12345678.cos.ap-shanghai.myqcloud.com
```

**获取方式**：
- 登录腾讯云控制台 → 对象存储 COS
- 创建存储桶或使用已有存储桶
- 在「访问管理 > API密钥管理」获取 SecretId 和 SecretKey

### 2. 预览/部署地址配置

修改 `application.yml` 中的预览和部署地址：

```yaml
app:
  deploy:
    # 已部署应用访问域名（需要能被截图服务访问）
    host: http://your-server-ip:8123/api/static
  preview:
    # 预览应用访问域名（需要能被截图服务访问）
    host: http://your-server-ip:8123/api/preview
```

**重要**：截图服务使用 Selenium WebDriver 访问这些地址，所以地址必须能从服务器本地访问。

### 3. Chrome/Chromium 浏览器

截图服务依赖 Chrome 浏览器，服务器需要安装：

**Windows**：
- 安装 Chrome 浏览器即可，WebDriverManager 会自动下载对应版本的 chromedriver

**Linux (CentOS/Ubuntu)**：
```bash
# Ubuntu
sudo apt-get install chromium-browser

# CentOS
sudo yum install chromium
```

或者安装完整的 Chrome：
```bash
# Ubuntu
wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
sudo dpkg -i google-chrome-stable_current_amd64.deb
sudo apt-get install -f
```

## 功能验证

### 检查配置是否生效

启动应用后，检查日志：
- `WebDriver 初始化成功` - 截图服务可用
- `COS未配置` - COS未配置，截图将仅保存在本地

### 手动测试截图

可以通过测试类 `LocalScreenshotExecutorTest` 验证截图功能：

```java
@Test
void testScreenshot() {
    String url = "http://localhost:8123/api/preview/html_1/index.html";
    CompletableFuture<String> future = executor.executeAsync(url);
    String path = future.get(60, TimeUnit.SECONDS);
    assertNotNull(path);
}
```

## 常见问题

### Q: 截图生成失败

检查以下项目：
1. Chrome/Chromium 是否安装
2. 预览URL是否能从服务器本地访问（用 curl 测试）
3. COS配置是否正确（检查 secretId/secretKey/bucket）

### Q: 截图封面没有更新

检查日志：
- `开始异步生成截图封面` - 截图任务已提交
- `应用封面更新成功` - 截图完成并上传

如果没有日志，可能是：
- appId 为空
- 截图服务不可用
- COS未配置

### Q: 本地开发环境截图不工作

本地开发时，COS可能未配置，截图会保存在本地临时目录：
```
tmp/screenshots/{uuid}/{random}_compressed.jpg
```

但不会上传，应用封面不会更新。
